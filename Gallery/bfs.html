Breadth Frist Search

As I have finally moved on from polar graphs, we're going to be covering a new subject, so I thought it would be appropriate to experiment: I made a video. In the video below, I'll be explaining the basic principles that this interactive program operates under. The program is used for pathfinding, so each square displays an integer called a pathlength. The pathlength just states how many steps it will take to get from the square the integer is contained in, to the destination square. This algorithm also accounts for boundaries, which can be constructed/deconstructed on the grid by right-clicking on the specified square, and changing destination squares which can be done by left-clicking. Squares that can't be reached are colored randomly and given a path-length of zero.
